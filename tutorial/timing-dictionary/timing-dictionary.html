Web Animations Tutorial

Web animations is to have a flash or GIF animation intergrated onto the web page. Normal web animations still require the use of plugins such as Java Applets, Shockwave, Adobe Flash. In this tutorial, we will be showing you how to create animations using javascript without the need of installing plugins.


Adding Timing Dictionary

Timing Dictionary is basically a set of timing properties being put together and apply for an animaton. Timing dictionary allows the grouping of all timing properties together to avoid rewrite of many lines of codes. There are a total of 9 memebers that can be included in a timing dictionary:

Note that when using the above properties, the duration of the animation will have to be defined under 'duration'. Also, properties with a type of 'double' as input should have a value greater than or equals to 0. A negative value would cause "Exception Errors".

The animation interface should be as follow when using timing dictionary:

new Animation(element, {effect}, {timing properties, duration: #});
  • startTime
    • Defines the start time of the animation. Defaulting to 'null'.
    • Value should be in double.
  • startDelay
    • Defines the number of seconds to wait before starting an animation.
    • Value should be in double.
  • fillMode
    • Defines the existence of effects outside of the active interval (i.e. the actual animation time) as outside of the active interval, a timed item may still have an effect.
    • There are 4 modes: none, forwards, backwards, both.
  • iterationStart
    • Defines the number of iterations into the timed item at which to begin.
    • For instance, a value of 0.5 woule cause the timed item to begin half-way through the first iteration.
  • interationCount
    • Defines the number of times to repeat an item. Defaulting to 1.0.
    • Value should be in double.
  • iterationDuration
    • Defines the duration of a single iteration. Defaulting to 'null'.
    • Value should be in double.
  • defaultPlaybackRate
    • A double number which multiplies to the inherited time potentially which causes the item to run at a different rate to its natural speed. Defaulting to 1.0.
    • Value should be in double.
  • direction
    • Defines the direction in which the animation proceeds. Defaulting to 'normal'.
    • It has 4 directions: 'normal', 'reverse', 'alternate', 'alternate-reverse'.
  • timingFunction
    • Defines the scaling of time to produce easing effects. Defaulting to 'null'. Further explaination in Timing Functions section.
